När vi började på uppgift 2 del 1 så började vi med att skapa variabler för att hålla koll på hur många lager ner i XML dokumentet vi kommit samt variabler som tex is_in_entryLinks för att kolla om någon av parent
taggarna hade varit en entryLinks tag. Men när vi jobbat på detta sätt ett tag och skapat flertalet variabler för att hålla koll på parent tags så kom vi på en bättre lösning. Det var att använda en array som en typ av stack. 
Det vi gjorde var lägga till i den arrayen varje gång man når en start tag och sedan ta bort den senaste när man når en sluttag. Detta gjorde så våran kod blev mycket mer lättläst och enkel att jobba med.
Något som detta också gjorde var att vi kunde väldigt enkelt konvertera våran stream parser till att använda XPath med en DOM parser. 

Vår SAX parser var i början väldigt komplicerad och vi behövde göra en hel del felsökning. Det som vi gjorde då var att vi debuggade det steg för steg. Kollar först om vi är i den tag vi förväntar oss att vara i sedan kollar vi 
dess parents och children. Det viktigaste är att dela upp problemet i mindre delar som man enklare kan testa.

Något som vi tyckte gjorde det enklare att förstå och tolka ett XML dokument var att använda de inbyggda funktioner i vs-code och flika ner datan så man bara kollar på ett lager åt gången.
